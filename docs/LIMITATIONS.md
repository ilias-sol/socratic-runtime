# Limitations

- Completion is Luna's source-level judgment, not executed or formally verified correctness.
- Model latency and quota depend on Codex CLI and the user's account.
- Live decisions are nondeterministic; exact intervention timing and wording may vary.
- Only one file is observed, so multi-file tasks may lack important context.
- VS Code diagnostics depend on installed language tooling.
- Language-neutral means one text-document contract, not equal evaluation quality in every language.
- The active-session leakage filter is intentionally conservative and can suppress a useful question.
- The prototype has automated engineering tests and a live simulated journey, but no controlled learner study yet.
